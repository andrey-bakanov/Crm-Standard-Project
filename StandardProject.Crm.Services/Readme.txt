Сборка предназначена для размещения типовой логики, которая может быть быть повторно использована
Каждый компонент представляется в виде сервиса
Сервис группирует методы, реализующие сценарий
Сервис может быть не привязан к отдельной сущности и может выступать как агрегатор
Сервис должен иметь конкретную функциональность и не иметь слишком общих методов
